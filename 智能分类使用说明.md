# 智能分类功能使用说明

## 功能概述

智能分类功能可以自动分析上传的文件，根据文件名和内容自动推荐最合适的分类。系统支持两种分类方式：

1. **规则引擎分类**（默认）：基于文件名和关键词的规则匹配
2. **AI智能分类**：使用AI服务进行智能分析（需配置API密钥）

## AI服务推荐

### 1. 百度文心一言（推荐）

**优势：**
- 中文理解能力强
- 价格实惠（约0.008元/千tokens）
- 国内访问稳定
- 教育领域效果好

**配置步骤：**
1. 访问 [百度智能云](https://cloud.baidu.com/)
2. 注册并创建应用
3. 获取 API Key 和 Secret Key
4. 在 `application.yml` 中配置：

```yaml
ai:
  enabled: true
  provider: baidu
  baidu:
    api-key: your-api-key
    secret-key: your-secret-key
    model: ernie-bot-turbo
```

**费用估算：**
- 每次分类约消耗500-1000 tokens
- 成本约0.004-0.008元/次
- 1000次分类约4-8元

### 2. 阿里云通义千问

**优势：**
- 阿里云生态集成好
- 支持长文本处理
- 性价比高

**配置步骤：**
1. 访问 [阿里云DashScope](https://dashscope.aliyun.com/)
2. 开通服务并获取API Key
3. 在 `application.yml` 中配置：

```yaml
ai:
  enabled: true
  provider: aliyun
  aliyun:
    api-key: your-api-key
    model: qwen-turbo
```

**费用估算：**
- 每次分类约消耗500-1000 tokens
- 成本约0.002-0.004元/次
- 1000次分类约2-4元

### 3. OpenAI GPT-3.5/4

**优势：**
- 理解能力最强
- 支持多语言
- 分类准确率高

**配置步骤：**
1. 访问 [OpenAI](https://platform.openai.com/)
2. 获取API Key
3. 在 `application.yml` 中配置：

```yaml
ai:
  enabled: true
  provider: openai
  openai:
    api-key: your-api-key
    model: gpt-3.5-turbo
    base-url: https://api.openai.com/v1
```

**注意事项：**
- 需要海外访问能力
- 价格较高（约$0.0015/1K tokens）
- 建议使用代理服务

### 4. 腾讯云混元大模型

**优势：**
- 国内服务稳定
- 价格适中
- 中文效果好

**配置方式：** 需要参考腾讯云官方文档

### 5. 智谱AI ChatGLM

**优势：**
- 开源模型可私有化部署
- 中文能力强
- 免费额度多

**适用场景：** 预算有限的项目

## 使用方法

### 前端使用

1. 登录系统，进入"资料上传"页面
2. 填写资料标题和描述
3. 开启"智能分类"开关
4. 上传文件
5. 系统将自动分析并推荐分类
6. 确认提交

### API调用

```bash
POST /api/material/upload
Content-Type: multipart/form-data

参数：
- title: 资料标题
- description: 资料描述
- file: 文件
- isPrivate: 是否为个人文件（0/1）
- enableAiClassify: 是否启用智能分类（true/false）
- categoryId: 分类ID（启用智能分类时可选）
```

## 规则引擎分类

即使不配置AI服务，系统也会使用规则引擎进行智能分类：

### 支持的规则

1. **文件名匹配**
   - 包含"试卷"、"考试" → 试题库
   - 包含"课件"、"ppt" → 课件
   - 包含"教案" → 教案
   - 包含"实验" → 实验指导
   - 包含"论文" → 学术论文
   - 包含"视频"、"mp4" → 教学视频

2. **关键词匹配**
   - "数据结构"、"算法" → 算法
   - "数据库"、"sql" → 数据库
   - "机器学习"、"深度学习" → 人工智能
   - "python" → Python编程
   - "java" → Java编程
   - "数学"、"微积分" → 数学
   - "物理"、"力学" → 物理

3. **默认分类**
   - 无法识别的文件 → 其他

## 文件内容提取

系统支持从以下格式提取文本内容：

- **PDF文件**：提取前10页文本
- **Word文档**：提取前50段落
- **PPT演示**：提取前10张幻灯片
- **Excel表格**：提取前50行数据
- **文本文件**：提取前100行

## 性能优化建议

1. **启用缓存**：对相似内容的分类结果进行缓存
2. **异步处理**：大文件采用异步分类
3. **批量处理**：对多个文件进行批量分类
4. **降级策略**：AI服务失败时自动使用规则引擎

## 故障排查

### 1. 分类失败

**可能原因：**
- AI服务API密钥配置错误
- 网络连接问题
- API额度不足

**解决方案：**
- 检查 `application.yml` 配置
- 查看后端日志
- 确认API账户余额

### 2. 分类不准确

**可能原因：**
- 文件内容过少
- AI模型选择不当
- 提示词需要优化

**解决方案：**
- 增加文件描述信息
- 尝试不同的AI模型
- 调整分类规则

### 3. 响应慢

**可能原因：**
- 文件过大
- AI服务响应慢
- 网络延迟

**解决方案：**
- 限制文件大小
- 使用更快的AI服务
- 启用异步处理

## 最佳实践

1. **混合使用**：规则引擎 + AI服务
2. **人工审核**：对重要文件进行人工确认
3. **持续优化**：根据分类结果调整规则
4. **监控成本**：定期检查API使用量
5. **备选方案**：准备多个AI服务提供商
