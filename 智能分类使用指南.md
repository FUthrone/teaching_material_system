## 💡 工作原理

### 智能分类流程

```
上传文件
   ↓
提取文件内容（PDF/Word/PPT/Excel/TXT）
   ↓
调用阿里云通义千问API
   ↓
AI分析文件名和内容
   ↓
返回最合适的分类名称
   ↓
自动匹配或创建分类
```

### 支持的文件格式

- **PDF文件**: 提取前10页文本
- **Word文档**: 提取前50段落
- **PPT演示**: 提取前10张幻灯片
- **Excel表格**: 提取前50行数据
- **文本文件**: 提取前100行

### 分类规则

系统采用**混合策略**：

1. **AI智能分类**（优先）
   - 分析文件名和内容
   - 理解语义和上下文
   - 推荐最合适的分类

2. **规则引擎**（降级）
   - 文件名关键词匹配
   - 内容关键词匹配
   - 文件类型识别

## ⚙️ 高级配置

编辑 `application.yml`:

```yaml

  aliyun:
    model: qwen-max  # 更强大的模型
```

### 调整内容提取长度

编辑 `FileContentExtractor.java`:

```java
// PDF提取页数
stripper.setEndPage(20);  // 默认10页

// Word提取段落数

// 文本提取行数
int maxLines = 200;  // 默认100
```

## 🔍 故障排查

### 问题1：分类失败

1. 确认后端服务已启动
2. 检查API Key是否正确

### 问题2：分类不准确

**优化方案：**
1. 增加文件描述信息
2. 使用更详细的标题
3. 切换到qwen-max模型

### 问题3：响应慢

**原因：**
- 文件过大
- 网络延迟
- AI服务响应时间

**解决：**
- 限制文件大小
- 使用缓存机制
- 异步处理
